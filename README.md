# MILP Cluster Analysis

**Десктопное приложение для кластерной линейной регрессии на базе смешанного целочисленного линейного программирования (MILP)**

</div>

---

## О проекте

Приложение разбивает набор наблюдений на *r* кластеров и строит **отдельную линейную регрессию** для каждого кластера так, чтобы суммарная абсолютная ошибка аппроксимации была минимальна. Разбиение и коэффициенты регрессии находятся **одновременно** — как решение единой MILP-задачи с помощью открытого решателя [HiGHS](https://highs.dev).

Поддерживается **многомерная регрессия**: в качестве предикторов можно выбрать один или несколько числовых столбцов из Excel-файла.

Основные возможности:

- **Загрузка Excel** (`.xlsx`, `.xls`, `.xlsm`, `.xlsb`, `.ods`) с выбором столбцов X и Y
- **Настраиваемое число кластеров** *r* (от 2 до 50)
- **Три режима размеров кластеров**: свободный, ручной (задание части размеров), поровну
- **Интерактивные Plotly-графики** кластерной модели и ошибок
- **Экспорт отчёта в PDF** с уравнениями, метриками и графиками
- **Локальное хранилище** (DuckDB): история наборов данных, результатов и PDF между запусками
- **Современный интерфейс** на PySide6 с темой qt-material (dark purple)

## Использование

| Шаг | Действие | Описание |
|-----|----------|----------|
| 1 | **Загрузить Excel** | Файл с минимум двумя числовыми столбцами |
| 2 | **Предпросмотр** | Выбрать столбцы X (предикторы) и Y (целевая переменная) |
| 3 | **Настроить параметры** | Число кластеров *r*, режим размеров кластеров |
| 4 | **Запустить расчёт** | MILP строит оптимальную кластерную регрессию в фоне |
| 5 | **Просмотреть результаты** | Графики, таблица коэффициентов, метрики |
| 6 | **Экспортировать PDF** | Отчёт сохраняется на диск и в локальное хранилище |
| 7 | **История** | Повторно открыть ранее импортированные наборы данных и PDF |

### Формат входных данных

Приложение читает **первый лист** Excel-файла. Нечисловые столбцы отбрасываются; строки с пропусками удаляются.

В диалоге предпросмотра:

- **Y** — ровно один столбец (целевая переменная);
- **X** — один или несколько столбцов (предикторы).

Примеры числовых наборов лежат в `data/sample_data.csv` и `data/sample_data_2.csv` — их можно открыть в Excel и сохранить как `.xlsx` для загрузки в приложение.

### Режимы размеров кластеров

| Режим | Описание |
|-------|----------|
| **Свободный** | Размеры кластеров определяются оптимизацией |
| **Ручной** | Задаются размеры первых *r − 1* кластеров; последний получает оставшиеся точки |
| **Поровну** | Все кластеры максимально одинакового размера |

## Локальное хранилище

При первом запуске создаётся каталог `storage/` с файлом `milp.duckdb`. В нём сохраняются:

- импортированные наборы данных;
- параметры и результаты расчётов;
- экспортированные PDF-отчёты.

Каталог `storage/` указан в `.gitignore` и не попадает в репозиторий.

## Математическая постановка

Пусть дано *n* наблюдений с вектором предикторов \(X_i \in \mathbb{R}^p\) и откликом \(y_i\), а также число кластеров \(r\).

Нужно найти:

- **бинарные переменные** \(\sigma_{ij} \in \{0,1\}\) — принадлежность наблюдения \(i\) кластеру \(j\);
- **коэффициенты** \(a_{0j}, a_{1j,f}\) линейной регрессии
  \(\hat{y}_i = a_{0j} + \sum_f a_{1j,f} \cdot X_{i,f}\) для каждого кластера \(j\);
- **вспомогательные переменные** \(u_i \geq 0\) — абсолютные ошибки аппроксимации.

Минимизируется суммарная абсолютная ошибка:

\[\min \sum_{i=1}^{n} u_i\]

При ограничениях:

- \(\sum_{j=1}^{r} \sigma_{ij} = 1\) — каждое наблюдение принадлежит ровно одному кластеру;
- линеаризация \(|y_i - \hat{y}_i|\) через Big-M и переменные \(u_i\);
- (опц.) фиксированные размеры отдельных кластеров.

В интерфейсе дополнительно показывается **средняя процентная ошибка**:

\[E = 100 \cdot \frac{\sum_i u_i}{\sum_i y_i}\]
